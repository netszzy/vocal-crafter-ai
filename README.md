# Vocal Crafter AI - 长篇语音合成流水线 (Open Source Edition)

本项目是一个基于 `Fun-CosyVoice3-0.5B` 构建的本地语音生成流水线，专为零样本音色复刻与有声书级别的大型长文本打造，包含直观的 WebUI 和一键批量命令行处理方案。

## 🌟 核心卖点

### 1. 🎙️ 极简音色复刻 (Zero-Shot Voice Cloning)
**只需 8~15 秒 的干净录音，即可完美复刻任何人的声音！**
> ⚠️ **【免责声明与法律警告】**：本功能仅供个人学习、技术研究与获得合法授权的配音创作使用。**严禁**将本工具用于任何未经授权的变声、伪造录音、电信诈骗、侵犯他人肖像权/名誉权/声音权等违法违规行为。使用者因滥用本工具所引发的一切法律责任与纠纷，均由使用者自行承担，与本项目及原作者完全无关！请务必遵守当地法律法规。
- **无需微调训练**：抛弃传统 TTS 繁琐的炼丹流程，随录随用。
- **高还原度**：凭借 CosyVoice3 强大的底层模型，仅需极短的参考音频，就能精确捕获原声的音色、语气与情感特征。
- **ASR 动态提取**：导入 10 秒音频，系统自动按需加载 Whisper 识别对应文本，提取完毕立刻释放显存，丝毫不影响后续长篇生成的性能。

### 2. 📚 工业级超长文本生成 (Long-form Audio Generation)
**专为长篇小说、博客、课程等万字级文本设计。**
- **智能长文切分**：一次性复制几千上万字的文本，系统会自动依据标点符号和设定的上限长度（如 120 字）进行智能切分，保证断句自然、不断气。
- **无缝无损拼接**：几十甚至上百个文本片段在生成完毕后，程序会自动合并成一个完整的极长 WAV 音频文件（可选设定每段停顿时间），拿来即用。
- **子进程稳定调度**：拒绝 WebUI 假死。TTS 推理被剥离在独立子进程中运行，就算生成一小时的有声书，内存和线程依然稳定如初。

## 🚀 快速启动

### 环境准备

建议使用 Python 3.10+，推荐在虚拟环境中安装：

```bash
# 核心依赖 (适配 Windows GPU)
pip install -r requirements-windows-gpu.txt
```

*(请确保您已经安装并配置好了对应的 CUDA 环境以及 `Fun-CosyVoice3` 核心库。如果是从零开始，可以参考 `docs/new-computer-deployment.md`)*

### 启动 WebUI (推荐)

日常使用强烈推荐直接启动可视化界面：

```bash
# 进入 ccwebui 目录
cd ccwebui

# Windows 一键启动脚本
start.bat
# 或
.\launch.ps1
```

在浏览器打开本地端口后：
1. 上传 `8-15秒` 的参考音频保存为新音色。
2. 将小说正文直接粘贴进生成框，点击执行。
3. 去泡杯咖啡，系统会自动切割、生成并拼装成一部有声书。

## 💻 命令行自动化批处理

若需结合其他程序流水线（如 `story-crafter-ai`）实现全自动运转，核心控制文件位于 `scripts/` 下。

### 1. 配置规则

修改 `scripts/batch_config.json`：
- `ref_wav`：`10秒`左右的参考音频路径 (存放在 `refs/clean/`)
- `ref_text`：参考音频里说的具体文字内容
- `input`：待生成的长文本 txt 文件路径
- `output_dir`：切分片段的临时输出目录
- `merged_output`：最终合成超长音频文件的输出路径

### 2. 执行流水线

```powershell
# Windows 环境下直接执行批处理
& ".\scripts\run_batch_from_config.ps1"
```

## 📁 核心目录结构

- `ccwebui/`：Gradio 网页端界面及所有后端调度逻辑。
- `CosyVoice/`：底层模型调用封装层 (请确保包含预训练模型)。
- `scripts/`：命令行批处理脚本和配置文件。
- `docs/`：详细部署和架构文档。
- `outputs/`、`refs/`、`参考音色/`：默认的临时输出与参考数据目录（开源版本仅保留结构）。
